## 2. All bug findings (by area)

### Backend core (app.py, rag_engine.py, brand_config.py)
- HIGH C1/C6/C7 — see above.
- MED `POSTCODE_RE` only accepts Dutch format (`1234 AB`); Belgian customers (we serve Belgium — `belgie_informatie.txt`) loop forever on "Dat lijkt geen geldige postcode". [app.py:313,1286]
- MED EN→NL translation overwrites the reformulated search query with a translation of the *original* query — reformulation + entity work discarded for English users. [rag_engine.py:604 uses `query`, should use `search_query`]
- MED KB re-ingestion skips *changed* files (only checks `<file>_chunk_0` existence) — edits to FAQ/product files are silently not indexed on persistent storage; works in prod only because Railway FS is ephemeral. [rag_engine.py:461,490]
- MED Default personality prompts contain literal `[BRAND_NAME]`, never interpolated. [brand_config.py:56,68]
- MED Embedding truncation `text[:8000*4]` chars can still exceed the 8191-token limit for Dutch text → API error. [rag_engine.py:148-152]
- MED Admin session secret falls back to the fixed string `admin-session::` when `ADMIN_API_KEY` is unset in debug → forgeable cookies in dev. [app.py:217]
- MED `_log_chat_message` does a non-atomic full-file rewrite (unlike the atomic `save_session_state`); crash mid-write corrupts the log, which is then *reset* (data loss). [app.py:886-895]
- MED Concurrent sends on one session do unlocked read-modify-write of the session file → lost state transitions (pairs with the missing frontend send-lock). [app.py:657-681]
- MED GDPR retention cleanup runs only at process startup — a long-lived container never deletes expired PII. [data_retention.py:62, app.py:449]
- LOW Language defaults inconsistent: `'en'` at app.py:934 and in `detect_language` no-client fallback vs `'nl'` everywhere else (Dutch company). [app.py:934, rag_engine.py:1138]
- LOW Unused import `get_shipment_status` in app.py. [app.py:20]
- LOW Rate limiter uses `memory://` — per-process limits with multiple workers. [app.py:160]
- LOW `max_completion_tokens=350` can cut answers mid-sentence. [rag_engine.py:1011]
